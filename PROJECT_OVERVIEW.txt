=== STREET DICE - ПОЛНОЕ ОПИСАНИЕ ПРОЕКТА ===

ДЛЯ ИГРОКА: В ЧЕМ ФИШКА?

Представь: ты бросаешь настоящие кости прямо в Telegram. Встряхиваешь телефон - кубики летят по столу с реалистичной физикой, стукаются, крутятся. Каждый бросок уникален.

Основной кайф:
1. РЕАЛЬНАЯ ФИЗИКА - кубики ведут себя как настоящие. Сила броска, угол, вращение - все имеет значение. Не рандом из генератора, а настоящая симуляция.

2. ИГРАЙ С ДРУЗЬЯМИ - создай лобби, пригласи друзей через Telegram. Все видят одни и те же броски в реальном времени. Соревнуйтесь, смейтесь над неудачами, празднуйте победы.

3. КАСТОМИЗАЦИЯ - твои кубики, твой стиль. Золотые, хромированные, неоновые, прозрачные. Столы от классического зеленого фетра до уличного асфальта. Создай свой уникальный сет в редакторе.

4. ЗАРАБАТЫВАЙ ИГРАЯ - каждый бросок приносит pips (внутренняя валюта). Используй бусты для x2, x3, x5 множителей. Копи на крутые кубики или ставь в играх с друзьями.

5. АЗАРТ СТАВОК - играй на pips с друзьями. Банк победителю. Блеф, риск, адреналин. Как в покере, но с костями.

6. РАЗНЫЕ РЕЖИМЫ:
   - Palmo's Dice: набери 200 очков первым, комбинации как в покере, но можешь сгореть
   - Poker Dice: 5 раундов покерных комбинаций, лучший выигрывает
   - Free Roll: просто кидай и зарабатывай pips

7. ПРИВОДИ ДРУЗЕЙ - получай редкие кубики и столы за активных рефералов. Чем больше друзей играет, тем круче награды.

Это не просто "нажми кнопку - получи результат". Это тактильный опыт: встряхнул телефон, увидел полет кубиков, услышал стук о стол, почувствовал вибрацию. Плюс социалка - играешь с живыми людьми, а не с ботами.

Короче: реалистичные кости + друзья + кастомизация + заработок + азарт ставок = затягивает.

ОБЩАЯ КОНЦЕПЦИЯ (для разработчика)
Многопользовательская 3D игра в кости для Telegram Mini App с реалистичной физикой, кастомизацией и монетизацией.

ТЕХНИЧЕСКИЙ СТЕК

Клиент:
- TypeScript + Vite
- Three.js (3D графика, рендеринг)
- Cannon-es (физический движок)
- WebSocket (реалтайм коммуникация)
- Telegram Web App SDK

Сервер:
- Node.js + TypeScript
- WebSocket (ws библиотека)
- PostgreSQL (Neon serverless)
- Grammy (Telegram Bot API)

АРХИТЕКТУРА

Клиент (src/):
- game/ - игровая логика, физика, рендеринг
  * Game.ts - основной класс игры, сцена, камера, физический мир
  * Dice.ts - класс кубика (3D меш + физическое тело)
  * ShakeDetector.ts - детектор встряхивания устройства
  * AudioManager.ts - звуки столкновений
  * DiceValidator.ts - валидация результатов броска
  * WallText.ts - отображение pips на стене
- multiplayer/ - сетевая синхронизация
  * WebSocketClient.ts - клиент WS, управление соединением
  * GameSync.ts - синхронизация игрового состояния
  * DiceSync.ts - синхронизация бросков кубиков (replay)
- ui/ - интерфейс
  * MultiplayerUI.ts - меню, лобби, друзья
  * BoostsModal.ts - модальное окно бустов
  * BettingModal.ts - модальное окно ставок
  * DiceEditorModal.ts - редактор кастомных кубиков
  * ReactionWheel.ts - колесо реакций
  * ConnectionIndicator.ts - индикатор соединения

Сервер (server/src/):
- websocket/ - WS сервер и обработчики
- services/ - бизнес-логика
  * auth.ts - аутентификация через Telegram
  * users.ts - управление пользователями
  * friends.ts - система друзей
  * lobby.ts - создание/управление лобби
  * game.ts - игровая логика
  * betting.ts - система ставок
  * boosts.ts - система бустов
  * referrals.ts - реферальная программа
  * telegram.ts - интеграция с Telegram (платежи, инвайты)
  * gameModes/ - игровые режимы
    - palmosDice.ts - режим "Palmo's Dice" (5 кубиков, комбинации)
    - pokerDice.ts - режим "Poker Dice" (покерные комбинации)
- db/ - работа с БД
- utils/ - утилиты (логирование, метрики, антифрод, rate limiting)

Общие модули (shared/):
- i18n/ - интернационализация (ru, en)
- presets.json - пресеты кубиков и столов

ФИЗИКА И ГРАФИКА

Физический движок (Cannon-es):
- Гравитация: -40 (ускоренная для динамики)
- Кубики: масса 0.5, размер 0.8, скругленные углы
- Материалы: dice, floor, wall с настройками трения/отскока
- Collision detection для звуков и вибрации
- Sleep mode для оптимизации

3D рендеринг (Three.js):
- PerspectiveCamera (FOV 60)
- WebGLRenderer с настраиваемым качеством (low/medium/high)
- Тени: PCFSoftShadowMap
- Освещение: AmbientLight + SpotLight + DirectionalLight
- Environment map для металлических отражений
- Процедурные текстуры (felt, leather, wood, marble, etc.)
- Реальные текстуры из манифеста (fabric_velour_velvet, wood_oak, etc.)

Настройки графики:
- Low: pixelRatio=1, без теней, без антиалиасинга
- Medium: pixelRatio=1.75, без теней, без антиалиасинга
- High: pixelRatio=2, с тенями, с антиалиасингом

Кастомизация кубиков:
- Цвет основы, точек, границ
- Материал: roughness, metalness, clearcoat
- Прозрачность
- Размер/форма точек (круг, квадрат, ромб)
- Глубина точек (normal map)
- Радиус скругления углов

Кастомизация столов:
- Отдельные настройки для пола и стен
- Текстуры: felt, leather, velvet, wood, marble, concrete, etc.
- Цвет, roughness, metalness, normal intensity
- Масштаб текстуры (small/medium/large)
- Реальные текстуры из файлов
- Декоративные бордюры

ИГРОВЫЕ РЕЖИМЫ

1. Free Roll (соло)
- Свободные броски без правил
- Начисление pips за броски
- Тренировочный режим

2. Palmo's Dice (2-6 игроков)
- 5 кубиков D6
- Цель: первым набрать 200 очков
- Каждый ход: бросок → выбор (взять очки или перебросить)
- До 3 бросков за ход
- Комбинации: пятёрка (100), каре (70), большой стрит (60), фулл-хаус (50), стрит (40), тройка (30), две пары (20), пара (10)
- СГОРАНИЕ: 2+ единиц на столе = -20 очков

3. Poker Dice (2-6 игроков)
- 5 кубиков D6
- 5 раундов
- Покерные комбинации
- Победитель раунда получает очко
- Побеждает игрок с наибольшим количеством раундов

МУЛЬТИПЛЕЕР

Лобби:
- Создание хостом (выбор режима)
- Присоединение по ID (8 символов)
- Голосование за стол (большинство побеждает)
- Отображение игроков с аватарами и экипировкой
- Инвайты друзьям через Telegram

Синхронизация бросков:
- Детерминированная физика с seed от сервера
- Запись траектории на клиенте бросающего
- Replay траектории у остальных игроков
- Синхронизация звуков столкновений
- Защита от читов (валидация результатов)

Система друзей:
- Поиск по username/nickname
- Отправка запросов в друзья
- Статусы: online, in_lobby, in_game, offline
- Инвайты в лобби

МОНЕТИЗАЦИЯ

Валюта - Pips:
- Начисляется за броски в соло (1-12 pips за бросок)
- Множители от бустов (x2, x3, x5)
- Бонусы от бустов (Snake Eyes: +250 за 1-1)
- Отображается на стене в 3D

Магазин (Telegram Stars):
- Кубики: common (0★), uncommon (70-90★), rare (100-220★), epic (150★)
- Столы: common (0★), uncommon (60-100★), rare (120-220★)
- Ключи для открытия кейсов

Система бустов:
- Double (x2 pips, 3 мин, кд 4 часа)
- Triple Even/Odd (x3 при чёт/нечёт, 3 мин, кд 4 часа)
- Snake Eyes (+250 за 1-1, 3 мин, кд 4 часа)
- Golden (x5 pips, 1 мин, кд 12 часов)
- Только один буст активен одновременно
- Хранение состояния в БД (user_boosts)

Система ставок:
- Ставки в pips на исход игры
- Минимальная ставка настраивается хостом
- Фазы: waiting → active → resolved
- Банк делится между победителями
- Защита: транзакции, валидация, rate limiting
- Возврат при дисконнекте до старта игры

Реферальная программа:
- Уникальный код для каждого игрока (DICE_XXXXXX)
- Награды за активность рефералов:
  * 1 друг сыграл 3 игры → случайный rare dice
  * 5 друзей сыграли 3 игры → случайный rare table
  * 10 друзей сыграли 3 игры → 3 случайных rare dice
- Награда за первую покупку реферала → предмет той же редкости
- Отслеживание в БД (referrals, referral_rewards)

БАЗА ДАННЫХ (PostgreSQL)

Основные таблицы:
- users: профили, баланс pips, экипировка, реферальный код
- item_catalog: каталог предметов (dice, table, effect)
- user_items: инвентарь игроков
- purchases: история покупок (Telegram Stars)
- friends: связи между друзьями
- friend_requests: запросы в друзья
- invitations: игровые инвайты
- lobbies: активные лобби
- lobby_players: игроки в лобби
- match_history: история матчей
- match_players: участники матчей
- user_boosts: активные бусты
- game_bets: ставки в играх
- referrals: реферальные связи
- referral_rewards: выданные награды

Индексы для производительности:
- users(telegram_id), users(nickname)
- friends(user_id), friends(friend_id)
- lobby_players(lobby_id), lobby_players(user_id)
- game_bets(lobby_id), game_bets(user_id)

ЗАЩИТА И БЕЗОПАСНОСТЬ

Антифрод:
- Валидация результатов бросков (физически возможные значения)
- Детерминированная физика с server seed
- Проверка скорости бросков (rate limiting)
- Логирование подозрительной активности
- Блокировка при обнаружении читов

Rate Limiting:
- 1 ставка в секунду
- Ограничение частоты сообщений
- Защита от спама

Транзакции:
- Атомарные операции с балансом
- Rollback при ошибках
- Проверка баланса перед списанием

Graceful Shutdown:
- Корректное завершение игр
- Возврат ставок
- Сохранение состояния
